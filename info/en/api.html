API

iDAI.chronontology provides a REST interface for querying and manipulating the data managed by our service. The base URL of the web service is http://chronontology.dainst.org/data.

HTTP Basic Authentication is used for all requests that require special priviliges.

All data is encoded in JSON.

The following operations are supported:


GET /period/

Performs a search for period data.

The following GET-parameters are supported:

  • q: Query term
  • fq: Filter query(repeatable)
  • from: Offset
  • size: Maximum number of results
  • facet: Attribute that will be used to generate a facet (repeatable)
  • exists: Only period terms that have this attribute will be returned (repeatable)
  • parts: Only the listed attributes will be returned (comma separated)

GET /period/:id

Return the metadata for the period term identified by :id.

The following GET-parameters are supported:

  • version: Allows accessing older versions of the record

PUT /period/:id

Updates or creates the record with the :id.

Expects a JSON representation of a period term in the body part of the request.


POST /period/

Creates a new period term with a random id.

Expects a JSON representation of a period term in the body part of the request.


DELETE /period/:deletedId/:replacementId

Marks the document specified by :deletedId as deleted and sets the document specified by :replacementId as its replacement.
