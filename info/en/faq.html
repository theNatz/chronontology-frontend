FAQ Frequently asked questions

Search

How to combine multiple keywords?

If you enter multiple keywords, every period term that reflects the content of all keywords, will be found. By default, keywords are linked with a logical AND.

Example: Bronze Age

How to exclude keywords?

By placing a hyphen in front of a keyword, you can exclude any period terms from the search result, which include the latter.

Example: Bronze Age -Levant

How to enter complex logical operators in a search query?

iDAI.chronontology supports the logical operators AND, OR and NOT. Pay attention to the usual operator precedence: NOT has priority to AND, and AND has priority to OR. You can use brackets to alter the precedence of the concatenation.

Example: Neolithic AND (Britain OR Europe)

How to search for entire phrases instead of multiple loose words?

Put the entered phrase between quotation marks. Hereby only objects that contain exactly this phrase can be found.

Example: "Reign of Romulus Augustulus"

Which wildcards are supported?

The search query can contain the following wildcards:

  • ? as any single symbol
  • * as none or multiple successive random symbols

Example: Roman*
(finds next to "Roman" terms like "Romano-German" or "Roman Tetrarchy")

Are regular expressions supported?

Yes, the search engine supports all common operators to formulate regular expressions. The latter must be placed between two slashes.

Example: /Pompe[ij]/
(finds results for “Pompeii” as well as for “Pompeji”)

The search query caused a syntax error, what to do?

Syntax errors may occur if the request contains reserved search operators (eg. `/` or `~`). If these operators are used within a search, they can be escaped with `\` or be used as part of a search phrase within `"`.

Example: 87\/243

What determines the order of search results?

By default search results in iDAI.chronontology are ordered by relevance. This depends on one hand on the relevance of an object in relation to the keywords. So objects appear earlier if they contain the keyword several times or in the title. On the other hand, the quality of a record flows into the relevance measurement. This gives highly contextualized and described records a better position in search results.

The default sorting can be changed right above the display of search results

Do filters show all available values?

No. Only the most common values for the specific search result are displayed

How to cite a record?

Every record has a permalink underneath its title, which should be used for citing it.

The link looks like this: chronontology.dainst.org/period/Gx4uxaeTCbbg
