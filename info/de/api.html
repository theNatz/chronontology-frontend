API

iDAI.chronontology bietet eine REST-Schnittstelle zur Abfrage und Manipulation der verwalteten Daten. Die Basis-URL des Webservice ist http://chronontology.dainst.org/data.

Die Authentifizierung erfolgt grundsätzlich über HTTP-Basic.

Als Austauschformat dient JSON.

Die folgenden Operationen werden unterstützt:


GET /period/

Führt eine Suche nach Epochenbegriffen aus.

Die folgenden GET-Parameter werden unterstützt:

  • q: Suchbegriff
  • fq: Filter (kann mehrfach verwendet werden)
  • from: Offset
  • size: Maximale Anzahl der Ergebnisse
  • facet: Eigenschaft für die eine Facette generiert werden soll (kann mehrfach verwendet werden)
  • exists: Nur Epochenbegriffe, die diese Eigenschaft besitzen werden zurückgegeben (kann mehrfach verwendet werden)
  • parts: Nur die aufgelisteten Eigenschaften werden zurückgegeben (kommagetrennt)

GET /period/:id

Gibt die Metadaten zum Epochenbegriff zurück, der über :id identifiziert wird.

Die folgenden GET-Parameter werden unterstützt:

  • version: Erlaubt den Zugriff auf ältere Versionen des Datensatzes

PUT /period/:id

Aktualisiert den Datensatz unter :id oder legt einen neuen Datensatz unter :id an.

Erwartet die JSON-Repräsentation eines Datensatzes im Body-Teil der Anfrage.


POST /period//h4>

Legt einen neuen Datensatz unter einer zufällig generierten ID an.

Erwartet die JSON-Repräsentation eines Datensatzes im Body-Teil der Anfrage.


DELETE /period/:deletedId/:replacementId

Markiert den Epochenbegriff mit der ID :deletedId als gelöscht und ersetzt durch den unter :replacementId angegebenen Epochenbegriff.
